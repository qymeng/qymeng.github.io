Hexo相关

博客目录结构

  • _config.yml，站点配置文件
  • scaffolds目录，存放了文章布局相关的模板文件，默认是post布局
  • source/_drafts是草稿目录，source/_posts是发表的博文目录
  • theme目录，存放主题文件
  • public目录，存放hexo根据md文件生成好的html静态网页

主要命令(点击查看官方说明)

new

新建一篇文章。layout，布局，比如有post（文章）、page（页面）、craft（草稿）。如果没有设置layout，默认使用_config.yml种的default_layout参数代替。如果标题包含空格，要使用括号括起来。

可简写为

publish

发布草稿。

可简写为

generate

生成静态文件。

可简写为

server

启动预览服务器。默认情况下，访问网址为：http://localhost:4000

可简写为

deploy

部署网站

新建layout

Hexo博客常用命令hexo n <title>去新建一篇文章，其实这是hexo new post <title>的简写，即以post为模板新建一个markdown文件。

我们可以新建模板，方法是在scaffolds中新建模板名.md，例如新建idea.md；

然后在该文件中设置好要设置front matter、以及其他模板内容；

然后就可以使用hexo n idea <title>来创建一篇idea。
