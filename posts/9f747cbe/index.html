Hexo博客多设备同步

发表于2022-03-17|更新于2022-03-17
|阅读量:

之前一直在macbook一台设备上写博客，最近又把之前的thinkpad用了起来，以后可能还会用到公司电脑、攒的windows主机等其他设备，于是面临着多设备同步的问题。

解决方法：

参考：https://www.jianshu.com/p/fceaf373d797

目录结构

.gitignore文件内容如下：

众所周知，public/目录下存放着博客生成的静态网页文件，也是部署到线上的文件，从.gitignore内容可以看出Hexo是支持我们把环境文件放到github上托管的，并且列好了忽略项。

怎么把环境文件托管到github

同一个仓库

  • github上新建一个hexo分支，git clone到本地
  • 仓库文件夹中都是静态网页，删除除了.git文件夹之外的所有内容，目的是获取一个空的hexo分支
  • git add -A，提交工作区变化。git add -A与git add .的区别是，前者包含已经删除的文件，后者不包含
  • git commit -m "some description"提交
  • git push origin hexo推送到远程，此时github上hexo分支已经被清空
  • 复制刚才的.git文件夹到Hexo博客根目录下。此时，Hexo项目已经变成了和远程hexo分支箱关联的本地仓库了。之前的user.github.io文件夹已经没用了，可以删除。
  • 以后每次修改博客内容（修改配置、发表博客）等，都要git add -A & git commit -m "some description" & git push origin hexo推送到远程。

不同仓库

  • 上面的方式有个问题，就是博客文件也会一同公开，个人不想这样。于是新建一个私有仓库托管博客文件。
  • 从上面的方式中可以看出，我们只要有一个内容为空的仓库的.git目录，然后把他复制到博客根目录下就可以把Hexo博客与远程仓库关联起来。于是偷懒不写了~

新电脑中的环境搭建

  • 安装node.js和git，windows上要注意添加环境变量
  • 安装hexo：npm install -g hexo-cli
  • clone远程仓库到本地git clone git@github.com:username/username.git.io.git -b hexo或者git clone git@github.com:username/MyBlog.giti
  • 运行npm install，会根据package.json安装依赖
  • 本地生成网页、启动博客服务器：hexo g & hexo s

多设备的同步操作

  • git pull从远程仓库拉取最新更新，尤其是在其他设备上更新过博客后
  • 文章写完后，推送博客文件git add -A & git commit -m "some description" & git push，发布博客hexo g -d。
